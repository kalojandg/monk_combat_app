## Таб 1: Инвентар

### Gold Tracker

Показва се горе, над таблицата с предмети.

**Текущо салдо** — показва PP / GP / SP / CP (read-only display).

**Gain** — добавяне на намерена валута:
- Полета: PP + GP + SP + CP (попълваш само тези, които намираш)
- Бутон **Gain** → добавя към текущото салдо без конверсия
- Пример: намираш "437 GP, 1946 SP, 9275 CP" → пишеш в полетата → Gain

**Spend** — харчене с автоматична конверсия:
- Полета: PP + GP + SP + CP (попълваш само това, което харчиш)
- Бутон **Spend** → изважда, като при нужда конвертира от по-висока деноминация
- Пример: харчиш 13 SP и нямаш SP → конвертира 2 GP → 20 SP → плаща 13 SP → остава 7 SP
- При недостатъчно средства: показва грешка, не прави промяна

**D&D конверсия:**
```
1 PP = 10 GP
1 GP = 10 SP
1 SP = 10 CP
```

**Spend алгоритъм** (чиста функция — лесно тестваема):
```
totalCP = pp*1000 + gp*100 + sp*10 + cp
costCP  = pp*1000 + gp*100 + sp*10 + cp
if totalCP < costCP → грешка
remainCP → разпредели обратно в PP/GP/SP/CP (minimize деноминации)
```

### Таблица с предмети
Колони: `☰` (drag handle) | Предмет | Брой | Тежест | Стойност | Носи | Бележки | Действия

- **Носи** — dropdown: Party / [имена на играчите — hardcoded в конфиг]
- **Действия** — Edit (молив) + Delete (кош)
- Footer ред: Общо тегло `X lb` | Обща стойност `X gp`

### Диалог — Добавяне / Редактиране
Отваря се с бутон "Добави предмет" или Edit икона.
Полета:
- Предмет (text, required)
- Категория (select): Оръжие / Броня / Консуматив / Разно
- Брой (number, default 1)
- Тежест (number, lb)
- Стойност (number, gp)
- Носи (select: Party / играчи)
- Бележки (textarea)

Бутони: **Запази** | **Отказ**

### Drag-and-Drop
- SortableJS на `<tbody>`
- Handle: `☰` иконата
- При drop: записва новия ред в Firestore

## Таб 2: Куестове

### Таблица с куестове
Колони: `☰` (drag handle) | Куест | Статус | Дадено от | Бележки | Действия

- **Статус** — badge с цвят:
  - `Активен` → жълт
  - `Изпълнен` → зелен
  - `Провален` → червен
  - `Паузиран` → сив
- **Действия** — Edit (молив) + Delete (кош)
- Бутон за бърз смяна на статус директно в реда (без да влизаш в диалога)

### Диалог — Добавяне / Редактиране
Отваря се с бутон "Добави куест" или Edit икона.
Полета:
- Куест (text, required)
- Статус (select): Активен / Изпълнен / Провален / Паузиран
- Дадено от (text — NPC/фракция)
- Описание (textarea — детайли, условия)
- Награда (text — xp, gold, итем)
- Бележки (textarea — бележки на групата)

Бутони: **Запази** | **Отказ**

### Drag-and-Drop
- SortableJS на `<tbody>`
- Handle: `☰` иконата
- При drop: записва новия ред в Firestore

## Споделени детайли

- Всяка промяна (add/edit/delete/reorder) се записва веднага в Firestore
- Firebase `onSnapshot` → real-time update при всички отворени клиенти без reload
- Firestore структура:
  ```
  /inventory/items        ← array или subcollection
  /inventory/gold         ← { pp, gp, sp, cp }
  /quests/items           ← array или subcollection
  ```
- Имената на играчите се дефинират в един `const PLAYERS = [...]` в горната част на `index.html`
